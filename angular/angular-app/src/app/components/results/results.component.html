• Request Id:  Every request have a unique Id​
  • Source: Source is the end node from where request raised​
  • Destination: Destination is the end node where source should be entangled​
  • Demand Size: Demand size is the no. Of entangled pairs requested between source & destination​
  • Start Time: Start time of the simulation ​
  • End Time: End time of the simulation ​
  • Memory Index: Every memory array has a unique Id​
  • Current Node: Node initiating request.​
  • Entangled Node: Destination Node.​ ​
  • Fidelity:  Fidelity is the quality parameter for the entanglement pair​ 0 < f < 1​
  • Entangled Time: Time of end to end entanglement made after start time of simulation ​
  • Expiry Time: Time of expiry of end to end entanglement after the entangled time​
  • State: Status of the memory​ Entangled- E2E​ Occupied – memory in use​ Raw- memory not in use​
